Why We Need to Change

Two Compounding Problems

01 Community Deprecation

The official kubernetes/ingress-nginx maintainer announced retirement of the community controller. End-of-life is March 2026 — no more security patches, no bug fixes, no upstream support.

Risk if we don't act
  • No CVE / security vulnerability patches
  • No upstream bug fixes or K8s compatibility updates
  • Breaks future Kubernetes version upgrades
  • Running unsupported software in production
02 Current Architecture Pain Points

Independent of the EOL, the existing setup carries its own operational debt that the migration is an opportunity to resolve.

Known issues today
  • Split annotation model — NLB config spread across ALB Controller and NGINX Ingress, fragile and confusing
  • Two controllers, one job — ALB Controller + ingress-nginx both required just to route traffic
  • Extra failure surface — NGINX ingress pods are an additional layer that can crash, OOM, or mis-route
  • Operational overhead — separate NGINX upgrade cycle, custom config, dedicated debugging path

The EOL deadline is a forcing function — but the architecture already benefits from simplification.

Our Decision

Why AWS Load Balancer Controller

01
Best fit for existing architecture

CloudFront → NLB → Services already relies on AWS-native components. AWS LBC integrates naturally — no infrastructure changes needed.

02
Lowest operational complexity

Removes ingress controller pods entirely. No NGINX-specific tuning, no controller upgrades, simplified troubleshooting.

03
Strong long-term support

Official AWS support, active maintenance, aligned with AWS networking roadmap. Avoids repeating the ingress-nginx EOL situation.

04
Already installed

Controller is deployed, IAM permissions established, team is familiar. Accelerates PoC and production timelines significantly.

05
Lower migration risk

Gateway API CRDs required but no new routing abstractions, no retraining required. Avoids major architectural change during a time-sensitive window.

06
CloudFront-centric design

WAF at CloudFront, CDN upstream, TLS at ALB — we don't need advanced L7 features. AWS LBC is sufficient without a full proxy layer.
